# Сутності домену: Магазин рибальської снаряги "FishHook Shop"

## Піддомени

- Каталог (Catalog)
- Замовлення (Ordering)
- Оплати (Payments)
- Доставка (Shipping)
- Користувачі (Users)

---

## 1. Піддомен: Каталог (Catalog)

### Item (Товар)

Опис: Окрема позиція рибальської снаряги, яку можна переглядати в каталозі та додавати в замовлення.

Атрибути:

- id: унікальний ідентифікатор товару
- name: назва товару
- description: текстовий опис
- price: ціна за одиницю
- categoryId: посилання на категорію
- stockQuantity: залишок на складі
- isActive: прапорець, чи активний товар у каталозі

### Category (Категорія)

Опис: Група товарів (наприклад, "Вудилища", "Котушки", "Приманки").

Атрибути:

- id: унікальний ідентифікатор категорії
- name: назва категорії
- description: опис категорії

---

## 2. Піддомен: Замовлення (Ordering)

### Order (Замовлення)

Опис: Замовлення користувача, яке містить один або більше товарів і переходить через різні статуси (чернетка, підтверджене, оплачене, відправлене).

Атрибути:

- id: унікальний ідентифікатор замовлення
- customerId: посилання на користувача, який створив замовлення
- items: список OrderItem
- status: поточний статус замовлення (draft, pending, paid, shipped, completed, cancelled)
- createdAt: дата і час створення
- updatedAt: дата і час останнього оновлення
- totalAmount: загальна сума замовлення

### OrderItem (Позиція замовлення)

Опис: Конкретний товар у межах одного замовлення.

Атрибути:

- itemId: посилання на товар (Item)
- name: назва товару на момент замовлення
- unitPrice: ціна за одиницю на момент замовлення
- quantity: кількість одиниць товару
- lineTotal: загальна сума за цю позицію

---

## 3. Піддомен: Оплати (Payments)

### Payment (Оплата)

Опис: Факт спроби чи здійснення оплати за замовлення.

Атрибути:

- id: унікальний ідентифікатор оплати
- orderId: посилання на замовлення
- amount: сума оплати
- status: статус оплати (pending, authorized, paid, failed)
- method: спосіб оплати (например, "cash_on_delivery", "card")
- createdAt: дата і час створення запису
- completedAt: дата і час завершення оплати (успіх або помилка)

---

## 4. Піддомен: Доставка (Shipping)

### Shipment (Відправлення)

Опис: Відправлення замовлення клієнту через службу доставки.

Атрибути:

- id: унікальний ідентифікатор відправлення
- orderId: посилання на замовлення
- trackingNumber: трек-номер відправлення
- status: статус відправлення (pending, prepared, shipped, delivered)
- deliveryMethod: спосіб доставки (наприклад, "nova_poshta")
- address: адреса доставки або дані відділення
- createdAt: дата створення запису
- updatedAt: дата оновлення статусу

---

## 5. Піддомен: Користувачі (Users)

### User (Користувач)

Опис: Учасник системи, який може бути покупцем або адміністратором.

Атрибути:

- id: унікальний ідентифікатор користувача
- email: електронна пошта
- name: ім’я або нікнейм
- role: роль користувача (customer, admin, manager)
- createdAt: дата реєстрації

---

## 6. Звʼязки між сутностями (високий рівень)

- Один User може мати багато Order.
- Один Order містить один або багато OrderItem.
- Кожен OrderItem посилається на один Item.
- Один Order може мати одну або кілька Payment (спроби оплати).
- Один Order повʼязаний з одним Shipment.
- Item належить до однієї Category.

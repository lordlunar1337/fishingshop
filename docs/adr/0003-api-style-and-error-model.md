# ADR 0003: Стиль REST API та модель помилок

- Статус: Accepted
- Дата: 2025-12-02

## Контекст

Потрібно описати контракт REST API для сутності `Item` у каталозі, реалізувати CRUD та єдиний формат помилок, який відповідає OpenAPI-специфікації.

## Рішення

- Стиль API: **REST**, транспорт — HTTP, формат даних — JSON.
- Базові операції для `Item`:
  - `POST /items` — створення товару.
  - `GET /items` — отримання списку товарів.
  - `GET /items/{id}` — отримання одного товару.
  - `PUT /items/{id}` — повне оновлення товару.
  - `DELETE /items/{id}` — видалення товару.
- Додатковий технічний endpoint:
  - `GET /health` — перевірка стану бекенду (повертає `200` і `"ok"`).

### Формат успішних відповідей

- Тіло відповіді повністю відповідає схемам `ItemResponse` та списку `ItemResponse[]` із `docs/api/openapi.yaml`.

### Формат помилок

Для всіх помилкових відповідей API повертає JSON обʼєкт формату `ErrorResponse`:

```json
{
  "error": "ValidationError",
  "code": "NAME_REQUIRED",
  "details": [{ "field": "name", "message": "Name is required" }]
}
```

# Roadmap: FishHook Shop

Цей документ описує план розвитку системи на найближчі тижні. Нумерація тижнів умовна та відповідає навчальному семестру.

## Тиждень 3

- Розширити модель товару:

  - додати поля ціни, опису, категорії
  - оновити структуру таблиці `items` у БД
  - додати початкові тестові дані з реалістичними товарами

- Оновити бекенд:

  - оновити endpoint `GET /items`, щоб повертати повну інформацію про товар
  - додати endpoint `GET /items/:id` для отримання детальної інформації

- Оновити фронтенд:
  - відображати ціну та категорію товарів у списку
  - додати сторінку або блок із детальною інформацією про обраний товар

## Тиждень 4

- Базова авторизація:

  - додати моделі користувачів у БД
  - реалізувати endpoints для реєстрації та входу користувача
  - реалізувати зберігання паролів у вигляді хешів
  - додати на фронтенді форми реєстрації та входу

- Розмежування доступу:
  - додати роль "адміністратор"
  - обмежити доступ до адміністративних операцій (керування товарами) тільки для адміністратора

## Тиждень 5

- CRUD для товарів (адмін-панель):

  - реалізувати endpoints:
    - `POST /items` — створення нового товару
    - `PUT /items/:id` — редагування товару
    - `DELETE /items/:id` — видалення товару
  - додати простий інтерфейс адмін-панелі на фронтенді для роботи з товарами

- Покращення каталогу:
  - фільтрація товарів за категорією
  - пошук за назвою

## Тиждень 6

- Кошик і замовлення:

  - реалізувати локальний кошик на фронтенді
  - реалізувати бекенд-логіку для створення замовлення
  - зберігати замовлення в БД
  - додати сторінку "Мої замовлення" для зареєстрованого користувача

- Підготовка до демонстрації:
  - базовий дизайн інтерфейсу
  - приведення в порядок документації в папці `docs/`
